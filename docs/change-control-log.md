This document serves as the authoritative, chronological log of all changes made
to the SOVERAIN Verification Bundle. Each entry records the version affected,
the type of change, the files modified, the reason for the change, and the
auditor or maintainer responsible.

All entries must be permanent, append-only, and timestamped.

---

## 1. LOG FORMAT REQUIREMENTS

Each change entry must include:

- Version Number  
- Change Type (MAJOR / MINOR / PATCH)  
- Description of Change  
- Files Modified  
- Reason for Change  
- Auditor or Maintainer  
- Timestamp  

No entry may be deleted or altered after publication.  
Corrections must be logged as new entries.

---

## 2. CHANGE TYPES

### MAJOR
Structural or protocol-level changes, including:
- new required files  
- removed required files  
- changes to verification logic  
- changes to ENS linkage requirements  
- changes to repository structure  

### MINOR
Non-breaking enhancements, including:
- new optional files  
- expanded metadata fields  
- documentation additions  
- QR metadata extensions  

### PATCH
Corrections or updates, including:
- checksum refresh  
- typo fixes  
- metadata timestamp updates  
- non-breaking documentation corrections
